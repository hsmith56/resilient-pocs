Runtime flow

  1. Read incident fields needed for routing: phase, impact rating, CBD, caused-by, type, owner, members, and lock type.
  2. If the current lock type is manually_set, stop immediately.
  3. Sort rules by descending priority and select the first matching rule.
  4. If the current lock type is condition_based but the current first match no longer sets a condition-based lock, clear the lock type and re-evaluate once.
  5. If no rule matches, preserve current owner and members.
  6. Apply the matched rule's owner, member changes, and lock type if configured.
