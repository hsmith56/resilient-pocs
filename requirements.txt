criteria 1
    in Triage phase:
        if incident.properties.cbd is missing or unknown, then set owner_id to "DISO-CIHT"
        if incident.properties.cbd == "GWM US", then set owner_id to "DISO-GWM US" and set a condition_based lock while this rule continues to match
        if incident.properties.impact_rating >= 3 and cbd is present, then set owner_id to the CBD business owner from CBD_BUSINESS_OWNER_MAP, defaulting to "DISO-CIHT", and set an incident_lifetime lock
        the incident_lifetime lock preserves the owner across phase, cbd, causedby, and type changes while impact_rating remains >= 3
        the incident_lifetime lock is cleared only when impact_rating drops below 3 or becomes blank/invalid
        if incident.properties.impact_rating < 3 and no higher-priority rule matches, then set owner_id to "DISO-CIHT"

criteria 2
    after Triage phase, including Response and Recovery and the rest of the incident lifecycle:
        if incident.properties.impact_rating >= 3 and cbd is present, then set owner_id to the CBD business owner from CBD_BUSINESS_OWNER_MAP, defaulting to "DISO-CIHT"
        if incident.properties.cbd is any of ["AM", "P&C", "GWM", "GWM WMI"] and incident.properties.impact_rating is 1 or 2, then set owner_id to the CBD business owner from CBD_BUSINESS_OWNER_MAP, defaulting to "DISO-CIHT"
        if incident.properties.cbd is any of ["GF", "IB"] and incident.properties.impact_rating is 1 or 2, then set owner_id to "DISO-CIHT"
        if incident.properties.impact_rating == 0, then set owner_id to "DISO-CIHT" unless a higher-priority rule matches
        members handling is currently disabled; the router does not add or remove members

criteria 3
    phase changes are handled by a Task rule named "Phase Change Check"
    when a task phase changes, the SOAR rule sets incident.properties.ownership_field_changed to True
    ownership_field_changed is included as a routing field so the Routing Field Changed rule runs full_script.py
    full_script.py resets incident.properties.ownership_field_changed to False before exiting
    phase changes do not override a manually_set lock or an incident_lifetime lock

criteria 4
    after Triage phase, if incident.properties.cbd == "GF" and incident.properties.impact_rating < 3:
        if incident.properties.causedby is "IT systems" or "Other third party", then set owner_id to "DISO-CIHT" and set a condition_based lock
        if incident.properties.causedby is "Employee" and incident.properties.type is "cyber attack" or "cyber incident", then set owner_id to "DISO-CIHT" and set a condition_based lock
    condition_based locks are cleared when no condition_based rule still matches, then routing is re-evaluated

criteria 5
    if incident.properties.assignment_owner_lock_type == "manually_set", full_script.py exits immediately and preserves the current owner_id
    manual_transfer.py currently sets incident.properties.assignment_owner_lock_type to "manually_set"

criteria 6
    if incident.properties.impact_rating == 0, then set owner_id to "DISO-CIHT" unless a higher-priority rule matches
    GWM US is higher priority and routes to "DISO-GWM US"

current audit behavior
    when full_script.py changes owner_id, it adds an incident note saying which routing rule changed the owner
